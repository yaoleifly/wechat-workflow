## 1. 安装与启动

### 环境要求

- Node.js 18 及以上版本

### 安装步骤

```bash
# 克隆项目
git clone https://github.com/yaoleifly/wechat-workflow.git
cd wechat-workflow

# 安装依赖
npm install

# 启动 Web UI
npm run serve
```

启动成功后终端会显示：

```
📝 公众号创作工作台
🌐 http://localhost:3210
```

用浏览器打开 `http://localhost:3210` 即可使用。

## 2. 配置模型与 API Key

首次使用需要完成模型和 API Key 的配置，配置信息会保存在浏览器本地，下次打开无需重新填写。

### 2.1 选择模型提供商

页面顶部有三个选项卡，点击切换提供商：

| 选项卡 | 说明 |
|--------|------|
| **Claude ⭐** | Anthropic 出品，默认推荐，写作质量最佳 |
| **OpenRouter** | 聚合平台，一个 Key 访问数百个模型 |
| **MiniMax** | 国内厂商，访问更稳定 |
| **Gemini** | Google 出品，多语言能力强 |

### 2.2 填写模型名称

选择提供商后，下方会显示「模型」输入框，预填了该提供商的默认模型名称。

- 可直接使用默认值，也可以修改为该提供商支持的任意模型名
- 常用模型参考：

  | 提供商 | 默认模型 | 其他可用模型示例 |
  |--------|----------|-----------------|
  | Claude | claude-opus-4-6 | claude-sonnet-4-6、claude-haiku-4-5-20251001 |
  | OpenRouter | anthropic/claude-opus-4-6 | openai/gpt-4o、google/gemini-2.0-flash-exp |
  | MiniMax | MiniMax-Text-01 | abab6.5s-chat |
  | Gemini | gemini-2.0-flash | gemini-1.5-pro、gemini-1.5-flash |

- 模型名称按提供商分别保存，切换提供商不会互相影响

### 2.3 填写 API Key

**获取 API Key**

点击 Key 输入框旁的 `?` 按钮，悬浮气泡会显示该提供商的 Key 获取步骤。快速入口：

- **Claude**：[console.anthropic.com](https://console.anthropic.com) → 左侧 API Keys → Create Key
- **MiniMax**：[platform.minimaxi.com](https://platform.minimaxi.com) → 右上角头像 → API 密钥 → 创建新密钥
- **Gemini**：[aistudio.google.com](https://aistudio.google.com) → 左侧 Get API Key → Create API key

**填写并验证**

1. 将复制的 Key 粘贴到输入框
2. 点击输入框旁的「显示」可切换明文查看，确认内容无误
3. 点击「验证并保存」按钮

系统会发起一次轻量测试请求验证 Key 是否有效：

- ✓ **已验证保存**（绿色）：Key 有效，已保存到浏览器本地，可以开始使用
- ✗ **API Key 无效，请检查后重试**（红色）：Key 有误，请检查是否复制完整、账户是否有余额

> Key 只保存在你自己浏览器的 localStorage 中，不会上传到任何服务器。

## 3. 运行工作流

### 3.1 输入选题

在配置区下方的输入框中填写选题，例如：

```
用 Tailscale 搭一个随身 AI 工作台
ZA Bank 开户全流程体验
Gemini 2.0 发布，对普通用户意味着什么
```

选题建议具体明确，方向越清晰，生成质量越高。

### 3.2 开始运行

点击「开始」按钮，工作流将自动依次执行 9 个步骤，每步结果实时流式输出。

页面顶部的进度条显示各步骤状态：

- 灰色：未开始
- 橙色闪烁：当前执行中
- ✓ 绿色：已完成，点击可查看该步结果

### 3.3 工作流 9 步说明

| 步骤 | 名称 | 内容 |
|------|------|------|
| Step 1 | 🎯 选题评估 | 从时效窗口、读者痛点、长尾价值、可写性四个维度评估选题质量 |
| Step 2 | 📦 素材准备 | 列出所需信息源、配图方案建议、可引用的工具和数据 |
| Step 3 | 🏗️ 结构搭建 | 搭建结论先行的文章大纲，含具体开头前三句话 |
| Step 4 | ✍️ 写作起稿 | 基于大纲写出 1500-2000 字完整初稿 |
| Step 5 | 🛡️ 敏感审查 | 逐段检查平台合规性，识别并修改敏感表述 |
| Step 6 | 🔍 事实核查 | 验证工具名称、产品功能、数据和技术描述的准确性 |
| Step 7 | 📡 传播优化 | 优化开头钩子、转发动机和结尾互动引导 |
| Step 8 | 💡 标题生成 | 生成 8 个备选标题，标注推荐项及理由 |
| Step 9 | 📄 格式输出 | 输出最终 Markdown 文章，附检查清单通过状态 |

### 3.4 暂停与继续

运行中点击「暂停」按钮，工作流会在当前步骤执行完毕后停止。

暂停后：
- 可以查看和编辑已完成步骤的内容
- 点击「继续」从暂停处接续执行

## 4. 查看与编辑步骤结果

### 4.1 切换查看步骤

点击进度条上任意已完成（绿色 ✓）的步骤，右侧面板切换显示该步结果。

### 4.2 编辑步骤内容

在步骤结果面板右上角点击「编辑」，内容变为可编辑的文本框，修改完成后点击「保存」。

> 编辑后的内容不会影响后续步骤的执行结果（后续步骤已经跑完），但保存文件时会使用编辑后的版本。

---

## 5. 保存与导出终稿

工作流全部完成后，Step 9 的结果即为最终交付版本。

### 5.1 复制终稿

在 Step 9 面板右上角点击「复制终稿」，内容复制到剪贴板，可直接粘贴到：

- [md.newkit.site](https://md.newkit.site) — 一键转微信公众号排版格式

### 5.2 保存为本地文件

点击「保存文件」，文章自动保存到项目目录下的 `output/` 文件夹：

```
output/
├── 2026-01-01_选题名.md          # 终稿
└── 2026-01-01_选题名_全流程.md   # 9 步完整记录
```

## 6. CLI 命令行模式

适合在终端操作或通过脚本调用。

### 6.1 配置 API Key

CLI 模式通过环境变量读取 Key（仅支持 Claude）：

```bash
export ANTHROPIC_API_KEY="sk-ant-xxxxx"
```

### 6.2 运行

```bash
# 交互式：启动后输入选题
npm start

# 直接传入选题
node cli.js "你的选题"
```

运行过程中会在终端实时流式显示每步进度和预览内容，完成后自动保存到 `output/` 目录。

## 7. 常见问题

**Q：验证 API Key 时提示「API Key 无效」**

- 检查 Key 是否复制完整，注意首尾不要有多余空格
- 确认账户有可用余额或额度
- Claude 的 Key 以 `sk-ant-` 开头；Gemini 的 Key 通常以 `AIza` 开头

**Q：运行中途报错中断**

- 检查网络是否正常
- 确认 API Key 的账户余额充足（9 步连续调用有一定消耗）
- 可以在报错步骤前手动暂停，编辑该步内容后继续

**Q：生成内容质量不理想**

- 选题描述更具体：加上背景、目标读者、核心角度
- 在 Step 3（结构搭建）完成后暂停，手动调整大纲，再继续后续步骤
- 修改 `workflow.js` 中的 `SYSTEM_PROMPT` 调整写作风格和账号定位

**Q：想换一个模型重新跑**

切换提供商/模型后，重新输入选题点「开始」（或点「重新开始」）即可。

**Q：输出文件保存在哪里**

项目根目录的 `output/` 文件夹，文件名格式为 `日期_选题名.md`。
